1

Listener vs Request

Two connectors look similar but face opposite directions. The HTTP Listener is inbound — it receives requests. The HTTP Request is outbound — your app becomes the client and calls someone else. The response replaces your payload.

2

The HTTP Request connector

It needs a method (usually GET) and a URL. For clean projects the base URL (host, port, protocol) lives in a shared request config, and each call sets just the path. We'll use a free, no-login test API: jsonplaceholder.typicode.com.

3

Call a real API

● Do this
  1. In your /hello flow, remove (or keep) Set Payload and add an HTTP Request after the listener.
  2. Create a request config with base URL host jsonplaceholder.typicode.com, port 443, protocol HTTPS.
  3. Set the request method to GET and path to /users.
  4. Save and re-run the project.
4

Test it

✓ Check yourself
  • Visiting http://localhost:8081/hello now returns a JSON list of users from the external API.
  • Changing the path to /users/1 returns just one user.
  • A Logger after the request prints the JSON in the terminal.
5

Stop hard-coding: properties

Real projects never bury URLs in the flow. Create a config.yaml in src/main/resources, put values like users.host: jsonplaceholder.typicode.com in it, load it with a Configuration properties element, and reference it as ${users.host}. This is exactly how the cert expects you to parameterize apps.
