1

Meet the Mule event

Everything moving through a flow is wrapped in a Mule event. It has three parts. The payload is the data itself. The attributes are read-only facts about it (like which URL was called). Variables are your own scratchpad you set as the flow runs.

More: the Mule event (official docs)

2

The HTTP Listener

A flow needs a source — the thing that starts it. The HTTP Listener is the most common one: it waits at an address and runs the flow whenever a request arrives. Its config has a host, a port (8081 by convention for local work), and a path.

3

Build a Hello World flow

● Do this
  1. Open your hello-mule project from Chapter 1 in VS Code.
  2. Open the main flow file under src/main/mule (it ends in .xml).
  3. Add an HTTP Listener as the flow's source. When asked, create a new listener config with host 0.0.0.0 and port 8081.
  4. Set the listener path to /hello.
  5. After the listener, add a Set Payload component and set its value to Hello World!.
  6. Save, then run the project (Command Palette → run/debug the Mule project).
4

Test it in your browser

Once the terminal says the app is deployed/started, open a browser.

✓ Check yourself
  • Visiting http://localhost:8081/hello shows Hello World!
  • Changing the Set Payload text and re-running changes what the browser shows.
  • No red errors appear in the VS Code terminal while it runs.
5

Peek at attributes

Add a Logger after Set Payload and set its message to #[attributes.method]. Re-run and watch the terminal when you hit the URL — it prints GET. That #[ ... ] is a tiny DataWeave expression reading the event's attributes. You'll use these constantly.
