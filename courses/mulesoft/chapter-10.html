1

From laptop to cloud

You manage deployed apps in Anypoint Runtime Manager. CloudHub (and CloudHub 2.0) is MuleSoft's hosted runtime: you upload your app, it runs on workers sized in vCores, and you get a public address plus logs and metrics.

More: CloudHub 2.0 (official docs)

2

Config that changes per environment

Dev, test, and prod need different values (URLs, credentials). That's why you externalized properties in Chapter 3 — at deploy time you set each environment's values, and you protect passwords with secure properties (encrypted).

3

Deploy it

● Do this
  1. Make sure you're signed in to Anypoint Platform from VS Code.
  2. Use the deploy action (Command Palette → deploy to CloudHub) and pick your environment.
  3. Give the app a unique name — it becomes part of the public URL.
  4. Set any required properties, then deploy and wait for status Started in Runtime Manager.
4

Test

✓ Check yourself
  • The app shows Started in Runtime Manager with no errors.
  • Its public URL (e.g. https://your-app.region.cloudhub.io/hello) returns your response from any browser.
  • You can open the app's logs in Runtime Manager.
5

Watch the bill

CloudHub uses paid vCores. On a trial you get limited free capacity — undeploy or stop apps you're not using so you don't burn through it.
