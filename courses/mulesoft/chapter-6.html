1

The three routers you must know

Choice picks ONE branch based on a condition (like if/else). Scatter-Gather runs several branches at the same time and gathers all results. For Each loops through a list one item at a time.

2

Choice in practice

A Choice has when branches with conditions and one otherwise fallback. Conditions are DataWeave expressions, e.g. #[attributes.queryParams.type == 'gold'].

3

Build a Choice flow

● Do this
  1. In your /hello flow, add a Choice router after the listener.
  2. Add a when that checks a query parameter, e.g. #[attributes.queryParams.vip == 'true'], and Set Payload to Welcome VIP.
  3. In otherwise, Set Payload to Welcome guest.
  4. Run it.
4

Test

✓ Check yourself
  • http://localhost:8081/hello?vip=true returns Welcome VIP.
  • http://localhost:8081/hello returns Welcome guest.
  • You can describe when you'd use Scatter-Gather vs For Each.
5

When to use which

Use Scatter-Gather when you need data from several systems and they don't depend on each other (faster, parallel). Use For Each when you must process items in order or one-by-one. Use Choice for branching logic.
