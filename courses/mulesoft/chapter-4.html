1

Adding a connector

Connectors beyond HTTP don't ship in the flow by default — you add them from Anypoint Exchange (MuleSoft's marketplace). Search the connector, add it, and it shows up as components you can drop into a flow.

2

The File connector

The File connector works with a folder on disk. Its key operations: a trigger, On New or Updated File, that starts a flow when a file appears; plus Read, Write, List, and Delete.

More: File connector (official docs)

3

Write then read a file

● Do this
  1. Add the File module from Exchange to your project.
  2. In a flow, after the listener, add a File Write operation. Set the path to output/greeting.txt and content to Hello from Mule.
  3. Add a second flow with a File Read of the same file, then a Logger to print it.
  4. Run and trigger both.
4

The Database connector

The Database connector runs SQL. It needs a database config pointing at your DB and a driver (a small library for that DB type). Operations map to SQL: Select, Insert, Update, Delete. A query returns rows as the payload — a list of records.

No database handy? You can practice the concept with a lightweight in-memory or local database — ask Claude Code to set one up for you (next station).

More: Database connector (official docs)

5

Test

✓ Check yourself
  • After running, output/greeting.txt exists on disk with the text you wrote.
  • The File Read flow logs the file contents to the terminal.
  • If you set up a database: a Select returns rows as a list payload you can see in a Logger.
