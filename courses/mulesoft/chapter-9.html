1

Why batch is different

Normal flows process one event end-to-end. A batch job splits a big collection into individual records and processes them in blocks, with built-in tracking of which succeeded and which failed. It runs in three phases.

More: batch processing (official docs)

2

The pieces

A batch job contains one or more batch steps (where you transform or send each record). A batch aggregator lets you handle records in groups (e.g. insert 100 at a time). The On Complete phase gives you counts of successes and failures.

3

Build a batch job

● Do this
  1. Add a flow that creates a list of sample records (e.g. a DataWeave list of 10 fake customers).
  2. Add a Batch Job scope and a Batch Step inside it.
  3. In the step, Log each record so you can watch them process.
  4. Add an On Complete that logs the success/failure counts.
  5. Run it.
4

Test

✓ Check yourself
  • The terminal logs each record being processed individually.
  • On Complete prints how many records succeeded.
  • You can describe the three batch phases from memory.
5

When to reach for batch

Use batch for large, record-oriented jobs (data migrations, nightly syncs) — not for a single API request. If a flow handles one message, keep it a normal flow.
