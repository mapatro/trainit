1

What integration actually means

Companies run many separate systems — a sales tool, a database, an email service, a payment provider. Integration is making them talk to each other reliably. MuleSoft is the tool in the middle that connects them, so you don't wire every system to every other system by hand.

2

APIs and HTTP — the language systems speak

An API is a doorway one system opens so others can ask it for data or actions. Most APIs talk over HTTP — the same protocol your browser uses. One side sends a request, the other sends back a response. Learn this picture once and most of MuleSoft clicks into place.

Two things worth memorizing: the common methods are GET (read), POST (create), PUT (update), DELETE (remove); and status codes tell you the outcome — 2xx success, 4xx you made a mistake, 5xx the server broke.

3

The MuleSoft vocabulary

You build a Mule app. Inside it are flows — step-by-step pipelines. Data travels through a flow as a Mule event. Each step is either a connector (talks to an outside system) or a core component (does logic, like transforming data).

Don't memorize every term yet — just recognize them. We build a real flow like this in Chapter 2.

Set up your workspace

Five installs. Anypoint Code Builder handles Java and Maven for you, so the list is shorter than it looks. (On Windows 11, just accept the defaults in each installer.)

● Do this
  1. Create a free Anypoint Platform account — 30-day trial, no credit card. Sign up here.
  2. Install VS Code — the editor everything runs in. Download.
  3. Install Git — for saving versions of your work. Download.
  4. Add the Anypoint Extension Pack in VS Code — open the Extensions panel (Ctrl+Shift+X), search Anypoint Extension Pack, install the full pack, then restart VS Code. Setup guide.
  5. Install Claude Code — your tutor in the terminal. Requires a paid Claude plan (Pro/Max) or API account. Install guide. The npm way: npm install -g @anthropic-ai/claude-code (needs Node.js 18+), or use the native installer from the docs.

Order matters: install Git and VS Code before the Anypoint Extension Pack. If the pack acts up, uninstall it, restart VS Code, and reinstall.

  • VS Code opens, and the Extensions panel lists Anypoint Extension Pack as enabled.
  • A MuleSoft icon appears in the VS Code left activity bar.
  • In a terminal, claude --version prints a version number.

6

First win — run a Mule app

Now the payoff: create a project and run it. Code Builder updates often, so menu wording may differ slightly from below — if it does, ask Claude Code (next station) or follow the official tutorial linked at the end.

● Do this
  1. In VS Code, open the Command Palette: Ctrl+Shift+P.
  2. Type Mule and choose the option to create a new Mule project (you can also start it from the MuleSoft icon in the activity bar).
  3. Give it a name like hello-mule and accept the suggested Mule runtime version.
  4. Wait for it to finish setting up — the first time pulls some files in the background. Keep the project's root folder open.
  5. Open the Command Palette again and choose Run / Debug the Mule project. Watch the terminal for a line that says the app is DEPLOYED or started.
✓ Check yourself
  • The project appears in VS Code with a pom.xml file and a src folder.
  • Running it shows a started / DEPLOYED message in the terminal with no red errors.
  • You can stop the app and run it again without it breaking.

If you'd rather build a tiny "Hello World" endpoint you can open in a browser, that's exactly what Chapter 2 does — one HTTP Listener returning a message. Getting the project to run cleanly today is the real Chapter 1 win.
