Every chapter has the same rhythm: a tight idea → a ● Do this → an AI prompt you paste into Claude/ChatGPT/Gemini to produce a real ClearEaty asset → a ✓ Check yourself. Do them in order — each builds the growth engine.

1

The few metrics that matter

Ignore vanity. Track: installs, activation (install → first scan), retention (D1/D7/D30), scans per active user, and referral rate. These tie straight back to your North-Star metric from Chapter 2.

2

Set up the tools

● Do this
  1. Use store consoles (App Store Connect + Play Console) for installs/listing metrics.
  2. Wire app events (Firebase/GA4) for first_scan, scan, share_score, retention — from the Ch.2 plan.
  3. Add UTM links to every campaign/post so you know which channel drove installs.
  4. Build one simple dashboard (a sheet is fine) pulling the 7 key numbers.

More: Get started with GA4

3

The weekly report

ai prompt · weekly report
› Act as a growth analyst for ClearEaty. Here are this week’s numbers [paste: installs, activation %, D1/D7 retention, scans/user, referral rate, by channel]. Write a tight weekly report: what changed vs last week, the 3 most important insights, what’s working/not by channel, and the single most valuable action for next week. Plain language, decision-focused.
✓ Check yourself
  • A 1-screen dashboard with the 7 key metrics
  • UTM tracking on all campaigns
  • A weekly report habit that ends in ONE next action

Measure activation and retention, not just installs — a free app’s value is in repeat use. Tools, prices and ad policies change fast — verify the current name, pricing, and rules before you rely on them. Never lock the work into one paid tool.
