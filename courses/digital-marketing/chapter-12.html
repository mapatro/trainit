Every chapter has the same rhythm: a tight idea → a ● Do this → an AI prompt you paste into Claude/ChatGPT/Gemini to produce a real ClearEaty asset → a ✓ Check yourself. Do them in order — each builds the growth engine.

1

What’s worth testing

Focus on the highest-traffic decision points: the store screenshots/subtitle, the landing hero, and the path to first scan. A win here multiplies every other channel.

2

Form hypotheses

ai prompt · test hypotheses
› Give me 8 prioritised A/B test ideas for ClearEaty to lift installs and first-scans. Cover: app-store screenshots/subtitle, landing-page hero, and onboarding-to-first-scan. For each: the hypothesis (“if we change X, then Y because Z”), the one variable to change, the metric to measure, and a rough effort/impact score. No medical claims in any variant.
3

Run one test properly

● Do this
  1. Pick the highest impact/effort idea; change exactly one variable (A vs B).
  2. Send comparable traffic to each and wait for enough data — don’t call it early.
  3. Decide in advance the single metric that declares a winner.
ai prompt · variant copy
› For my top test [describe A], write 3 “B” variants changing only one element (e.g. the hero headline OR the first screenshot caption). Keep ClearEaty’s honest, non-medical voice. Predict which might win and why, so I know what I’m really testing.

Beware false wins from tiny samples. If unsure whether a result is real, verify with a significance calculator. Tools, prices and ad policies change fast — verify the current name, pricing, and rules before you rely on them. Never lock the work into one paid tool.

4

Read it & bank the win

ai prompt · interpret results
› Here are my A/B test results [paste numbers: variant, visitors, conversions]. Tell me which performed better, whether the difference looks meaningful or could be noise, what to do next (ship/iterate/re-test), and one follow-up hypothesis.
✓ Check yourself
  • A prioritised test backlog (8+ ideas)
  • One test live with a single changed variable
  • A pre-declared success metric
  • A documented decision: ship, iterate, or drop
